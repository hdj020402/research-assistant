1. 科研前沿跟踪
	1. 自动从 arxiv 收集相关领域的文章
	   参考 https://github.com/hdj020402/chemrxiv-weekly 。chemrxiv-weekly 的功能是每周从 ChemRxiv 上根据关键词检索最新的文章，通过 GitHub Actions 开启定时任务。在当前项目中，可以借助该工具作为 agent，并开发 arxiv-weekly 工具（可以在该工具的基础上开发，也可以检索 GitHub 获取现成的代码，这类项目应该有很多）。
	   对于 arxiv-weekly，由于我的专业是化学，因此我的关注点更多在 AI for Chemistry 上，而 arxiv 上的文章/关键词更多侧重于算法等方面，因此需要设计合理的方法来检索符合我的领域的文章。
	   chemrxiv-weekly 还需要完善，具体的完善方案需要进一步讨论，但优先级不高。
	2. 从邮箱中获取订阅的期刊定期发送的文章链接
	   对于大多数期刊，如 Springer / ACS 出版社旗下的期刊，应该是没有提供类似 arxiv 那样的 api 来让用户检索文献的，因此我现在的做法是订阅了部分期刊，这些期刊会定时将新的文献推送到我的邮箱。通常来说，每次推送都会包含 10-20 篇甚至更多的文献，我需要一篇一篇点开链接去阅读摘要（可能还会因为网络原因卡很久），而其中可能只有小部分文献是真正能给我带来启发的。
	   在这一部分，我希望能设计一个工具，自动将推送邮件中的文章下载下来，提取摘要信息汇总给我（也可以不下载文章，直接在文献页面获取摘要，因为有些文章是付费/需要订阅的，下载起来有些难度，可以使用第 3 部分提到的文献下载工具（待开发））。
	   这一部分的难点在于，如何获取我的邮箱中的邮件，如何读取邮件内容，是否能保证使用过程中的安全。
2. 领域深挖
	这一部分主要是希望针对一个特定的领域进行深度调研，例如找到该领域的开山之作，获取一些关键节点取得的关键进展，收集那些影响力很大的工作，从而得到该领域的总览，明确哪个方向是值得进一步深挖的。
	在我的印象中，openai 好像推出过一个名叫 deepresearch 的科研调研工具，是否可以直接使用或借鉴？
	我在文献调研的时候用过 Semantic Scholar（[Semantic Scholar | AI-Powered Research Tool](https://www.semanticscholar.org/)），在这个网站上，可以通过关键词模糊检索的方式检索到某一领域的相关文献，并且在特定文献的页面中，还会罗列该文献的参考文献以及 cited by 信息，即以该文献作为一个节点，向上溯源并向下展开，能够获取该领域发展的完整脉络。然而问题也出在这里，如果每篇文献都有 10 篇相关的 ref 以及 10 篇相关的 cited by，那么到第二层就是 100 篇相关的 ref 以及 100 篇相关的 cited by，作为一个注意力和精力有限的人很难处理那么多信息，因此迫切需要借助 AI 的帮助。
	此外，考虑到科研总是在不停地发展，在对某一领域进行过一次深挖后，还是需要定时（e.g. 每周一次）跟踪该领域最新的进展，这就需要与任务 1 结合。
3. 文献整理
	1. 文献下载
	   借助如 Scihub 之类的网站，输入文献的 doi 号下载。下载时需要注意控制操作的时间间隔，避免被认为是在非法爬取信息，导致 ip 被封。
	   可以参考 GitHub 上是否有比较成熟的项目，这样就不需要从头实现了。
	2. 与 zotero 联用
	   zotero 有网页版插件，可以很方便地在文献页面中通过点击插件图标将文献加入到指定目录，方便整理。且撰写论文的时候也需要通过 zotero 来管理参考文献，因此最好能够将看过的文献都加到zotero中
	3. 依托 obsidian 管理
		1. 日常文献
		   详情页文件夹层级：`date_range/journal/article.md`
		   检索页：使用 obsidian 的 dataview 功能创建一个多列的表格，包括但不限于 | 文献标题 | 摘要（中英文） | 期刊 | 影响因子 | 发表日期 | 领域（tag 形式，可以有多个）| 双链指向详情页的 `article.md` |
		   对于详情页的 `article.md`，可以考虑创建一个 obsidian 格式的模板，每一篇文章都按照模板在规定地方填入对应的内容。（可以用 skills 实现？）
		   对于检索页的 dataview，每一列的信息都是从详情页中抽取出来的，所以只需要写 dataview 的代码即可。我希望在里面包含摘要的原因在于需要快速浏览文章大意，进而判断该文章是否对我的研究有用，可以精读。但是考虑到 obsidian 对于表格的优化做得不是很好，如果在表格中包含大量文字可能会导致表格的形状变得很奇怪，那就舍弃这一点，让用户直接点击双链到详情页中阅读即可。
		2. 领域深挖文献
		   详情页文件夹层级：`field/number_article.md`
		   这一部分的文献就不是仅仅收录摘要那么简单了。由于需要了解开山之作、关键节点的关键进展等信息，需要将这些影响力大的文献全都下载下来，并借助 AI 工具对这些文章进行拆解、分析。最基础的例如文献讨论了一个什么样的问题，用了什么方法来解决，得到了什么结果；进阶一点的例如该文献还遗留了什么未解决的问题。这些内容可以设计一套专门的 prompt 来由大模型生成，并且都需要按照规定模板写入详情页当中。
		   这一部分同样需要检索页，但我希望可以有一个类似 obsidian 自带的关系谱图这样的界面（或是思维导图？如果生成 xml 格式比较方便也行，可以生成 drawio 适配的 xml，由用户自行复制生成），领域的开山之作作为最大的节点置于页面上方，并不断地向下开枝散叶，每个子节点可能关注的方向都不一样，但是大方向都是符合该领域的。
	4. 科研 idea 头脑风暴
	   对于日常文献，大部分文献内容可能都和我自己目前的课题关系不大，但是这些课题一定也都是有意义的，例如我可以从中获取下一个新课题的灵感。在我的印象中，近期有发布过一个自动科研的工具，即大模型自主完成论文构思、实验、撰稿的全流程工作，不需要人类的介入。这一工具目前主要是用在 AI 会议领域，我在想是否也可以用到化学科研领域，并且目前我的主要诉求就是通过头脑风暴产生新的 idea。当然，这一部分并不是很急切的需求，毕竟目前我手头还有挺多进行中的课题。
	5. 文献汇报 ppt 的制作
	   每个月我都需要进行一次文献汇报，针对近期阅读的一篇文献，向课题组的老师和学生介绍文章内容。在大模型的帮助下，阅读文献已经不是难事，但是制作 ppt 却很耗费时间。Google 推出的 NotebookLM 可以用来辅助制作 ppt，但是我使用之后感觉效果并不是很好，例如无法详细且完整地列出文章中重要的部分，输出的是无法修改的 pdf 等。我了解到 Claude Code 有针对 ppt 制作的 skills，结合上述那么多文献调研相关的工作，我觉得可以将文献汇报 ppt 的制作也自动化完成。

该项目涉及众多功能，其中有些功能必须要借助大模型实现，例如对于整篇文献的分析；有些功能则可以通过编写 agent 工具，调用 agent 实现，例如文献下载。我希望尽可能提高 agent 工具的占比，尽可能减少大模型 token 的消耗。
